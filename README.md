# Копия Fund Dashboard (локально)

Это локальная копия интерфейса сайта:
`https://fund-dashboard-gilt.vercel.app/`

Сделано так:
- интерфейс (вкладки, таблицы, графики, стили) скопирован 1-в-1 в `public/index.html`;
- локальный сервер `server.js` поднимает собственный API;
- сборщик `data-provider.js` берет данные напрямую с `investfunds.ru`, считает доходности и агрегаты;
- desktop-режим запускается через Electron и открывает отдельное окно как приложение на Mac.

## 1) Установить Node.js (если еще не установлен)

Проверка:

```bash
node -v
npm -v
```

Если команд нет, поставьте Node.js LTS (рекомендую 20+) с официального сайта:
https://nodejs.org/

## 2) Установить зависимости

```bash
npm install
```

## 3) Запуск в режиме приложения (окно на Mac)

```bash
npm run app
```

## Запуск в 1 клик (как вы просили)

В папке проекта создан файл приложения:
`Аналитика фондов.app`

Просто нажмите на него 2 раза в Finder.
- Терминал не откроется.
- Откроется окно приложения.
- Иконка Electron в Dock на Mac скрыта по умолчанию.
- Если Node.js не установлен, появится понятное окно с подсказкой.

## 4) Запуск в браузере (если нужно)

```bash
npm start
```

После запуска открыть:
http://localhost:3000

## Важно про первый запуск

- На первом запуске сервер собирает данные и строит локальный кэш.
- Это может занять несколько минут.
- Дальше приложение стартует быстрее (используется кэш-файл в `.cache/`).
- Повторные обновления ускорены: данные по истории догружаются инкрементально, а карточки фондов переиспользуются из кэша.

## Тюнинг скорости (опционально)

Можно управлять скоростью загрузки через переменные окружения:
- `LIST_CONCURRENCY` (по умолчанию `8`) — параллельность загрузки страниц списка.
- `DETAIL_CONCURRENCY` (по умолчанию `16`) — параллельность загрузки карточек фондов.
- `HISTORY_BATCH_SIZE` (по умолчанию `48`) — сколько фондов в одном batch-запросе истории.
- `HISTORY_BATCH_CONCURRENCY` (по умолчанию `6`) — сколько batch-запросов истории выполнять параллельно.
- `HISTORY_LOOKBACK_DAYS` (по умолчанию `14`) — глубина догрузки истории при инкрементальном обновлении.
- `HIDE_DOCK_ICON` (по умолчанию `1`) — скрывать иконку из Dock (`0` — показывать).
